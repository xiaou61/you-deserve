# Progress

## 2026-05-06
- 已继续打磨个人中心与复习模式的“稍后题回收”体验，不再只给一个整批入口，而是补成多条可直接开刷的恢复路径。
- `/me` 的“上轮稍后题清单”现在新增 4 个细分动作：`只收明显卡壳题`、`先收有笔记卡点`、`按反复浏览优先`、`先收你收藏过的`，都直接落到 `slugs` 范围复习。
- `/review` 现在会把 `slugs` 范围明确显示成“题单范围：N 题”，并补一段题单说明，避免用户误以为队列异常变短。
- `/me` 新增“帮你整理好的复习题包”区块，当前包含 `临考冲刺包`、`笔记回捞包`、`旧收藏清仓包`、`最近回看包`、`回温巩固包` 五种直接可开刷的题包。
- 题包区已继续增强为轻任务编排：新增 `30 分钟快刷包`、`本周重点包`、`路线止损包`，并为每个题包显示预计阅读时长。
- 题包入口已和 `/review` 链路打通：支持通过 `bundle` 查询参数把题包名称带入复习页，让题单复习时能明确显示来源。
- `/review` 的完成收尾区已继续增强：当用户刷空某个题包后，会根据当前学习状态主动推荐下一包，而不是只停在“本轮完成”。
- 题包完成状态已回流到个人中心：复习轮次现在会记录 `bundleName`，`/me` 会显示“你刚清掉了哪个包”以及下一包建议。
- 题包列表已加入轻量优先级衰减：刚清完的包会自动降一点优先级；如果这轮还留了稍后题，则保留回收提示，不会简单消失。
- 题包区已继续增强为生命周期视图：每个包现在会显示 `新出现 / 值得接手 / 刚完成 / 还有尾巴 / 暂时空闲` 等状态，并提供整体状态计数。
- 题包区已继续增强为路径面板：每个包现在会显示“通常下一步会接哪一包”，并直接提供 1-2 个推荐续接入口。
- 题包区已继续增强为路径回放：个人中心现在会显示最近几次真实发生过的 `A -> B` 任务衔接路径，而不是只保留一条最新提示。
- 路径回放已继续增强为习惯总结：个人中心现在会归纳“你最常见的接包路径”和“你最常开的题包”。
- 题包区已继续增强为偏好纠偏：如果系统发现用户总在轻量包里打转、很少接重包，或者已有尾巴未收，就会主动给出纠偏提示和对应入口。
- 本轮工程验证已完成：`npm run lint` 通过，`npm run build` 通过，构建产物继续包含 `/me` 与 `/review`。
