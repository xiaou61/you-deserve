# 双非上岸八股学习站设计

## 目标

做一个面向双非本科同学的面试八股学习网站。第一阶段重点不是在线 AI 生成，而是把 AI 辅助打磨后的内容以本地 Markdown/MDX 的方式沉淀到仓库里，再用网站提供清晰、好看、移动端友好的学习体验。

## 产品定位

- 公开内容用于搜索引擎收录和自然传播。
- 登录能力后续加入，用于收藏、掌握状态、复习进度。
- 内容更新走开发流程：用户给资料或方向，开发者生成/优化内容文件，然后重新部署。
- 网站运行时不接入 AI，不暴露 AI Key，也不承担在线生成成本。

## 技术路线

- 框架：Next.js App Router
- 语言：TypeScript
- 样式：Tailwind CSS
- 内容：本地 Markdown/MDX 文件 + frontmatter
- Markdown 渲染：unified/remark/rehype 管线
- 图标：lucide-react
- 未来数据库：PostgreSQL + Prisma，仅存用户、收藏、进度、统计
- 部署：Vercel，未来数据库可选 Supabase 或 Neon

## 内容模型

每道题是一个 Markdown/MDX 文件，frontmatter 描述元信息：

- title：题目标题
- slug：URL 标识
- category：分类
- tags：标签
- difficulty：难度
- route：学习路线
- scene：面试场景
- order：排序

正文统一为：

- 一句话结论
- 通俗解释
- 面试回答
- 追问
- 易错点
- 记忆钩子

## 首版页面

- 首页：学习路线、搜索、分类筛选、推荐题目、学习进度预览。
- 题目详情页：左侧/顶部题目信息，中间正文，底部相关题目。
- 路线页预留：后续按 Java 后端、前端、计基等路线扩展。

## 设计方向

视觉上避免普通资料站的廉价感，采用“清爽但有战斗感”的学习界面。移动端优先，电脑端提高信息密度。页面直接进入学习，不做空泛营销首页。

## 后续阶段

1. 第一阶段：内容站、MDX 题库、响应式页面、静态搜索。
2. 第二阶段：登录、收藏、掌握状态、复习列表。
3. 第三阶段：运营数据后台，统计浏览、收藏、高频搜索词。
