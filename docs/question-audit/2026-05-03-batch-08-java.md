范围：排序后的第 71-80 道题，`content/questions/java/arraylist-fail-fast.md` 至 `content/questions/java/concurrenthashmap.md`

## 总体结论

这一批 Java 题的基础答案整体可用，尤其 `arraylist-linkedlist`、`cas-aba`、`checked-runtime-exception`、`completablefuture` 的人工正文质量较好，已经能支撑一面/二面基础追问。主要问题不是大面积事实错误，而是自动扩写段落把 Java 语法/并发题强行套进工程化模板：

- 10 篇都出现了“数据量或并发量扩大 10 倍怎么办”“灰度、压测和回滚策略”“配置不生效”“JFR/GC 日志”等泛化表达，和集合、异常、BigDecimal、CompletableFuture 的题目焦点不匹配。
- `atomic-classes-cas`、`bigdecimal-precision`、`completablefuture-exception` 存在重复 `## 图解提示`。
- 多个图解节点有截断省略，如 `ArrayList 的 fail-fast 是什…`、`BigDecimal 使用时要注意什么…`、`CompletableFuture 异常怎么处…`，生成图会显得不完整。
- `atomic-classes-cas`、`bigdecimal-precision`、`completablefuture-exception` 的 `常见追问` 明显像占位模板，只把正文短句复制成回答，应该优先补成真实追问。

## 优先修复建议

1. 先清理 Java 批次的通用工程化模板，把追问改成集合/并发/异常/数值计算的专项问题。
2. 优先补 `atomic-classes-cas`、`bigdecimal-precision`、`completablefuture-exception` 的 `常见追问`，这三篇当前最像自动占位。
3. 合并重复 `图解提示`，并修复所有 `…` 截断节点，尤其是 BlockingQueue、CompletableFuture、ConcurrentHashMap 这类需要流程图表达的题。
4. 并发题建议统一增加“适用边界”：CAS 高竞争成本、BlockingQueue 容量、CompletableFuture 线程池隔离、ConcurrentHashMap 复合操作非天然原子。
5. 保留 `arraylist-linkedlist`、`cas-aba`、`checked-runtime-exception`、`completablefuture` 的优质正文，只针对追问和图解做专项化，不需要大面积重写。
