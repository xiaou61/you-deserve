## 一句话结论

LRU 通常用哈希表 + 双向链表实现。哈希表负责 O(1) 定位节点，双向链表负责维护最近使用顺序。

## 通俗解释

LRU 像桌面只放有限本书。你刚看过的书放最前面，很久没看的书慢慢被挤到最后。桌面满了，就把最后那本最久没看的拿走。

哈希表让你快速找到书，链表让你快速调整书的位置。

## 面试回答

设计思路：

1. 哈希表 `Map<key, Node>` 存 key 到链表节点的映射。
2. 双向链表按最近使用顺序排列，头部最新，尾部最旧。
3. `get` 时，如果 key 存在，把节点移动到头部。
4. `put` 时，如果 key 存在，更新值并移动到头部；如果不存在，插入头部。
5. 超过容量时，删除尾部节点，并从 Map 中移除。

这样 `get` 和 `put` 都可以做到 O(1)。

## 常见追问

### 为什么要双向链表？

删除一个已知节点时，需要快速连接它的前后节点。单链表不知道前驱节点，删除不方便。

### Java 里有没有现成结构？

可以用 `LinkedHashMap` 实现 LRU，它内部维护了链表顺序。

## 易错点

- 不要只用数组或普通队列，删除中间节点会慢。
- 不要忘记更新已有 key 时也要移动到头部。
- 不要忘记容量超限时同步删除 Map。

## 记忆钩子

**LRU 两件套：Map 负责找得快，双链表负责挪得快。**
