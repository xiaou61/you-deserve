## 一句话结论

稳定排序指相等元素排序前后的相对顺序不变。不稳定排序可能打乱相等元素原来的顺序。

## 通俗解释

按成绩排序时，两个同分同学原来 A 在 B 前面。稳定排序后 A 仍在 B 前面；不稳定排序可能把 B 放到 A 前面。

## 面试回答

稳定性在多关键字排序中很有用。比如先按姓名排序，再按成绩稳定排序，成绩相同的人仍保持姓名顺序。

常见稳定排序有冒泡排序、插入排序、归并排序。常见不稳定排序有选择排序、快速排序、堆排序。

## 常见追问

### 这题最容易被追问的边界是什么？

重点说清空输入、重复元素、极值、无解、越界、负数、溢出和复杂度退化。不要只给结论，要说明哪些条件下结论成立，哪些条件下会退化或需要换方案。

### 怎么证明自己不是在背模板？

用边界样例、时间复杂度、空间复杂度、单调性、递归深度和反例做验证。能说出具体信号、反例或排查入口，答案就从概念层进入实战层。

### 和相近方案怎么区分？

可以拿暴力解、排序、哈希、双指针、二分、堆、动态规划或图遍历对比，从正确性、复杂度、边界样例和实现难度几个维度选择。

### 面试官继续深挖时怎么展开？

先围绕“稳定性在多关键字排序中很有用。比如先按姓名排序，再按成绩稳定排序，成绩相同的人仍保持姓名顺序。”讲清主链路，再补“把 Java 相关表述再精确一点：对象数组排序稳定，基本类型数组不需要保留对象相对顺序”。如果能把边界条件、异常分支和验证闭环连起来，就能接住二面、三面的追问。

## 易错点

- 不要把稳定性理解成性能稳定。
- 不要说所有高级排序都稳定。
- 要结合相等元素相对顺序解释。

## 详细讲解

排序算法的稳定性是什么意思？ 这道题现在要从“能背出来”修到“能接住追问”。核心结论是：理解相等元素排序前后相对顺序是否保持，以及稳定排序的应用场景。回答时先把它放回算法语境，说明它解决什么矛盾，再围绕输入规模、状态定义、边界样例和复杂度推导展开。这样能避免泛泛而谈，也能让面试官听出你知道这题的真实边界。

第一步是拆清问题背景。按成绩排序时，两个同分同学原来 A 在 B 前面。稳定排序后 A 仍在 B 前面；不稳定排序可能把 B 放到 A 前面。在算法题里，背景不能写成空泛的工程套话，而要落到先给暴力思路，再说明为什么当前数据特征允许优化，最后落到状态、指针、堆、哈希或递归栈这些可验证结构。如果只说“看场景”或“加组件”，读者很难知道下一步该检查什么；如果能把输入、状态、依赖和输出说清，答案就有了主线。

第二步是讲机制。可以围绕这些点展开：稳定性在多关键字排序中很有用。比如先按姓名排序，再按成绩稳定排序，成绩相同的人仍保持姓名顺序。；常见稳定排序有冒泡排序、插入排序、归并排序。常见不稳定排序有选择排序、快速排序、堆排序。。每个点都要回答三个问题：它在链路里的位置是什么，它改变了什么状态，失败或边界条件下会留下什么现象。把这三件事讲清楚，追问从概念跳到实战时就不会断。

第三步是补边界。这里尤其要主动说明空输入、重复元素、极值、无解、越界、负数、溢出和复杂度退化。不要把稳定性理解成性能稳定。；不要说所有高级排序都稳定。；要结合相等元素相对顺序解释。很多八股答案的问题不是方向错，而是没有说适用范围；一旦遇到数据规模变化、异常分支或版本差异，原来的结论就可能不成立。

第四步是补专项深度：把 Java 相关表述再精确一点：对象数组排序稳定，基本类型数组不需要保留对象相对顺序。这部分适合放在面试回答后半段，用来证明你不仅知道标准答案，也知道高频追问会往哪里挖。

最后收束成一套回答节奏：先给结论，再讲机制，再补边界，最后说验证方式。对这题来说，验证可以看边界样例、时间复杂度、空间复杂度、单调性、递归深度和反例；方案选择可以和暴力解、排序、哈希、双指针、二分、堆、动态规划或图遍历对比。这样既保留八股题的清晰度，也能把答案讲成可落地、可排查、可复盘的经验。

如果面试官继续追问，可以再补一层取舍：为什么不用另一个方案、什么时候会退化、怎么发现反例或异常、出问题后如何止血。算法题最怕只有名词没有链路，补上输入规模、状态定义、边界样例和复杂度推导之后，答案会更稳。
