## 一句话结论

Invisible Index 是让索引暂时对优化器不可见，常用于评估删除索引是否会影响查询，而不是真的立刻删掉索引。

## 通俗解释

它像把工具先收进柜子但不扔掉。看看大家没有这个工具能不能正常干活，不行就立刻拿出来。

## 面试回答

索引太多会增加写入成本和存储成本，但直接删除索引有风险。不可见索引提供了一个灰度方式：

- 把某个索引设为 invisible。
- 优化器默认不再选择这个索引。
- 观察慢 SQL、执行计划和业务指标。
- 如果没影响，再考虑真正删除。
- 如果有影响，快速改回 visible。

它适合索引治理和性能优化验证，尤其在不确定索引是否被重要查询依赖时很有用。

## 常见追问

### invisible index 会节省存储空间吗？

不会。索引仍然存在并维护，只是优化器默认不使用它。

### 和直接 drop index 有什么区别？

drop 是删除，恢复需要重建；invisible 是隐藏，可以快速恢复可见。

## 易错点

- 不可见索引仍有写入维护成本。
- 不要把它当成空间优化手段。
- 变更后要观察执行计划和慢 SQL。
