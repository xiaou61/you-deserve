## 一句话结论

覆盖索引是指查询需要的字段都能从索引中拿到，不需要回表查询完整行，因此可以减少 IO。

## 通俗解释

你查资料时，如果目录上已经写了你要的信息，就不用再去档案柜拿整份档案。这个目录就覆盖了你的查询。

## 面试回答

如果 SQL 查询的 select 字段和 where/order by 等字段都包含在同一个索引中，数据库可以直接从索引返回结果，不需要再根据主键回表。

EXPLAIN 的 Extra 中出现 `Using index`，通常说明使用了覆盖索引。

覆盖索引能减少回表次数，提升查询性能，尤其适合高频查询和分页场景。

## 常见追问

### 覆盖索引是不是字段越多越好？

不是。索引字段越多，维护成本和存储成本越高，还会影响写入性能。要围绕高频查询设计。

### select * 会影响覆盖索引吗？

通常会。因为 select * 需要所有字段，索引很难覆盖全部字段，容易回表。

## 易错点

- 不要为了覆盖索引盲目建超宽索引。
- 不要在高频查询里随手 select *。
- 不要把 Using index 和 Using index condition 混为一谈。

## 记忆钩子

**覆盖索引就是目录信息够了，不用翻档案。**
