## 一句话结论

抽象类更适合表达“是什么”的继承关系和代码复用；接口更适合表达“能做什么”的能力规范。

## 通俗解释

抽象类像“动物”这个父类，猫狗都是动物，可以共享一些基础属性。接口像“会飞”这个能力，鸟、飞机、无人机都可以实现。

## 面试回答

抽象类可以有成员变量、构造方法、普通方法和抽象方法，适合抽取共同状态和行为。Java 只能单继承，所以一个类只能继承一个抽象类。

接口主要定义行为规范。Java 支持一个类实现多个接口。接口从 Java 8 开始可以有默认方法和静态方法，但它仍然更偏能力抽象。

## 常见追问

### 什么时候用接口？

当你要定义一组能力，让多个不相关类都能实现时，用接口更合适。

### 什么时候用抽象类？

当多个类有明显父子关系，并且需要共享状态或基础实现时，用抽象类更合适。

## 易错点

- 不要只背语法差异，要说设计语义。
- 不要忘记 Java 支持多接口实现。
- 接口有默认方法后，也不代表它和抽象类完全一样。

## 记忆钩子

**抽象类讲身份，接口讲能力。**
