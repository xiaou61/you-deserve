## 一句话结论

HashMap 默认负载因子 0.75 是空间利用率和查询性能之间的折中，太小浪费内存，太大容易冲突。

## 通俗解释

储物柜不能刚放一半就换大柜，也不能塞到快爆才换。0.75 就像柜子用到七八成时扩容，比较均衡。

## 面试回答

负载因子决定 HashMap 什么时候扩容：

```text
threshold = capacity * loadFactor
```

默认容量 16、负载因子 0.75 时，阈值是 12。元素超过阈值后会扩容。

如果负载因子太小，扩容更频繁，空间浪费更多。如果负载因子太大，数组利用率更高，但哈希冲突概率上升，链表或红黑树上的查找成本可能增加。

所以 0.75 是一个经验折中。

## 易错点

- 负载因子不是越大越好。
- 扩容阈值不是固定值，而是容量乘负载因子。
- 优化 HashMap 常先设置初始容量，而不是乱改负载因子。
