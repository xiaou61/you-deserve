## 一句话结论

反射允许程序在运行期获取类信息、创建对象、访问字段和调用方法。它让框架更灵活，但也有性能、安全和可维护性成本。

## 通俗解释

普通代码像提前知道门牌号再敲门。反射像到了现场才查这栋楼有哪些房间、每个房间能不能进去，还能动态调用里面的人做事。

## 面试回答

反射常用能力包括：

- 获取 Class 对象。
- 获取构造器、方法、字段。
- 动态创建对象。
- 动态调用方法或访问字段。

Spring、MyBatis、JUnit 等框架都大量使用反射来实现对象创建、依赖注入、注解解析等能力。

缺点是性能低于直接调用，可能破坏封装，也更容易在运行期才暴露错误。

## 常见追问

### 获取 Class 对象有哪些方式？

常见有 `类名.class`、`对象.getClass()`、`Class.forName("全类名")`。

### 反射为什么慢？

它需要运行期解析元数据、做访问检查，调用路径也比直接方法调用更复杂。

## 易错点

- 不要把反射说成只用于破坏 private。
- 不要忽略框架使用反射的价值。
- 不要滥用反射写业务逻辑。

## 记忆钩子

**反射就是运行时照镜子：看自己是谁，还能动态做事。**
