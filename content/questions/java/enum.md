## 一句话结论

Java 枚举适合表达固定取值集合，比如订单状态、用户角色、支付渠道，比字符串常量和数字常量更类型安全。

## 通俗解释

枚举像一份固定菜单。用户只能从菜单里点菜，不能随便手写一个不存在的菜名。

## 面试回答

枚举的优点：

- 取值范围固定，类型安全。
- 可读性比数字常量更好。
- 可以定义字段、构造器和方法。
- 适合配合 switch 或状态判断。

项目里常用枚举表达订单状态、消息状态、错误码、业务类型等。相比直接使用字符串，枚举能减少拼写错误和非法值。

## 常见追问

### 枚举能不能有构造方法？

可以，但构造方法默认是私有的，枚举实例在类加载时创建。

### 枚举适合存数据库吗？

可以存枚举 code，而不是直接存枚举名称，避免名称调整影响历史数据。

## 易错点

- 不要把枚举当成普通字符串常量。
- 数据库存储要注意 code 的稳定性。
- 枚举太多业务逻辑时，也要注意职责边界。

## 记忆钩子
