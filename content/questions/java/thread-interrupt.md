## 一句话结论

Java 选择协作式中断，线程是否退出取决于任务代码是否检查中断状态并正确处理 InterruptedException。

## 通俗解释

interrupt 像举黄牌提醒，不是直接把人拖下场。

## 面试回答

可以从这几层回答：

- 调用 interrupt 会设置线程中断标记。
- sleep、wait、join 等阻塞方法收到中断会抛 InterruptedException，并清除标记。
- 业务循环应定期检查 Thread.currentThread().isInterrupted()。
- 捕获 InterruptedException 后通常要恢复中断标记或明确退出。

回答时最好补一句：这个点不是孤立知识点，真正落地时要结合业务场景、数据规模和失败兜底，说明你不是只背概念。
