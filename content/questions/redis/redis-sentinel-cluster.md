## 一句话结论

Redis 主从复制解决数据副本和读扩展，哨兵在主从基础上做故障发现和自动切主，Cluster 通过分片解决更大规模的数据和访问压力。

## 通俗解释

主从像一个店长带多个店员，哨兵像巡检员发现店长倒了就选新店长，Cluster 像多家门店按区域分摊客流。

## 面试回答

三者可以分层理解：

- 主从复制：主节点负责写，从节点复制数据，可承担读请求或备份。
- 哨兵模式：监控主从节点，主节点故障时自动选举新主节点。
- Cluster：把数据按槽分布到多个主节点，每个主节点可以有从节点，实现分片和高可用。

主从和哨兵适合数据规模没那么大、主要需要高可用的场景。Cluster 更适合数据量和吞吐都更高的场景，但客户端路由、跨槽操作会更复杂。

## 常见追问

### 哨兵能解决数据分片吗？

不能。哨兵主要解决监控和自动故障转移，不负责把数据拆到多个主节点。

### Cluster 为什么有 16384 个槽？

Redis Cluster 用槽来映射 key 到节点，节点扩容迁移时移动槽即可，不需要逐个管理所有 key。

## 易错点

- 不要把哨兵和 Cluster 混为一谈。
- Cluster 不支持随意跨槽多 key 操作。
- 高可用不等于数据永不丢，主从复制仍可能有延迟。

## 记忆钩子

**主从管复制，哨兵管切主，Cluster 管分片。**
