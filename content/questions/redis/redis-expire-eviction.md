## 一句话结论

过期删除处理的是 key 到期后怎么删；内存淘汰处理的是 Redis 内存达到上限后删哪些 key。它们不是一回事。

## 通俗解释

过期删除像食品到了保质期要下架。内存淘汰像货架满了，必须挑一些商品拿走给新商品腾位置。

一个看时间，一个看空间。

## 面试回答

Redis 过期删除主要有：

- 惰性删除：访问 key 时发现过期再删除。
- 定期删除：后台定期抽样检查并删除过期 key。

只靠惰性删除可能导致过期 key 长期占内存，只靠定期删除又可能消耗太多 CPU，所以 Redis 结合两者。

内存淘汰是在达到 maxmemory 后，根据策略选择 key 淘汰，比如 LRU、LFU、随机淘汰、只淘汰设置过期时间的 key 等。

## 常见追问

### Redis 为什么不扫描所有过期 key？

全量扫描成本太高，会影响主线程处理请求，所以采用定期抽样 + 惰性删除的折中策略。

### LRU 和 LFU 区别？

LRU 看最近是否使用，LFU 看使用频率。热点数据场景下 LFU 可能更合理。

## 易错点

- 不要把过期删除和内存淘汰混为一谈。
- 不要说 Redis 到期一定立刻删除。
- 不要忽略 maxmemory 配置。

## 记忆钩子

**过期看时间，淘汰看空间。**
