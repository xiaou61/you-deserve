## 一句话结论

Redis 快主要因为数据在内存里、采用高效数据结构、使用 IO 多路复用，并且单线程避免了大量锁竞争和线程切换成本。

## 通俗解释

Redis 像一个动作很快的前台，资料都放在手边，不用跑仓库。虽然只有一个人处理主要命令，但他不用和别人抢笔、抢本子，也不用频繁交接。

## 面试回答

Redis 单线程指的是命令执行主流程主要由单线程处理，不代表 Redis 完全没有其他线程。

它快的原因：

- 纯内存操作。
- 高效数据结构。
- IO 多路复用，一个线程处理多个连接。
- 避免多线程锁竞争和上下文切换。
- 命令设计大多比较轻量。

但如果执行大 key 操作或慢命令，仍然可能阻塞主线程。

## 常见追问

### Redis 后来有没有多线程？

新版本引入了多线程处理网络 IO 等能力，但命令执行仍然要关注单线程模型下的阻塞风险。

### 单线程是不是不会有并发问题？

单条命令执行具备原子性，但多个命令组合仍可能有并发问题，需要事务、Lua 或分布式锁等机制。

## 易错点

- 不要说 Redis 完全单线程。
- 不要忽略慢命令和大 key 的阻塞。
- 不要把单命令原子性扩展到多命令组合。

## 记忆钩子

**Redis 快：数据在手边，一个人处理，不抢锁。**
