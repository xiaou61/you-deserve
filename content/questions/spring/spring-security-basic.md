## 一句话结论

认证解决“你是谁”，授权解决“你能做什么”。Spring Security 通过过滤器链在请求进入业务代码前完成安全检查。

## 通俗解释

进公司先刷工牌，这是认证；刷完工牌后能不能进机房、财务室，这是授权。

很多同学把登录成功和拥有所有权限混在一起，这是不对的。

## 面试回答

认证是确认用户身份，比如用户名密码登录、短信登录、Token 校验。授权是在身份确认后，判断这个用户是否有权限访问某个资源或执行某个操作。

Spring Security 的核心在过滤器链。请求进入 Controller 前，会先经过一系列安全过滤器，完成认证信息提取、身份校验、异常处理和权限判断。

在前后端分离项目里，常见做法是登录后发放 Token，后续请求携带 Token，服务端解析并构建认证上下文，再做权限判断。

## 常见追问

### Session 和 JWT 有什么区别？

Session 通常由服务端保存登录状态，客户端保存 session id。JWT 把部分身份信息放在令牌里，服务端可以无状态校验，但要注意过期、撤销和泄露风险。

### 权限控制放前端可以吗？

前端可以做展示控制，但真正的权限校验必须在后端做，因为前端逻辑可以被绕过。

## 易错点

- 不要把认证和授权混为一谈。
- 不要认为 JWT 天然更安全。
- 不要只在前端做权限控制。

## 记忆钩子

**认证问“你是谁”，授权问“你能去哪”。**
