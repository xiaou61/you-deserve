## 一句话结论

Spring AOP 用来把日志、事务、权限、监控这类横切逻辑从业务代码中抽出来，通过代理在方法执行前后织入增强逻辑。

## 通俗解释

业务方法像主线剧情，日志、权限、事务像每集都要出现的固定流程。如果每个方法里都手写一遍，代码会很乱。

AOP 就像在主线剧情外面套一层流程模板：进入方法前检查权限，执行时记录耗时，出错时回滚事务。

## 面试回答

AOP 是面向切面编程，解决的是横切关注点复用问题。Spring AOP 底层主要基于代理实现：如果目标类有接口，通常可以用 JDK 动态代理；如果没有接口，可以使用 CGLIB 生成子类代理。

常见概念包括：

- Join Point：连接点，比如方法执行。
- Pointcut：切点，定义哪些方法需要增强。
- Advice：通知，定义增强逻辑。
- Aspect：切面，切点和通知的组合。

Spring 事务就是 AOP 的典型应用。

## 常见追问

### JDK 动态代理和 CGLIB 区别？

JDK 动态代理基于接口生成代理对象；CGLIB 基于继承生成目标类子类。final 类或 final 方法会影响 CGLIB 代理。

### AOP 为什么会有自调用失效？

因为同类内部 `this` 调用不会经过代理对象，增强逻辑就不会执行。

## 易错点

- 不要把 AOP 说成替代 OOP，它是补充。
- 不要忘记 Spring AOP 主要作用于 Spring Bean。
- 不要忽略代理导致的自调用问题。

## 记忆钩子

**AOP 就是给业务方法套流程外壳：进门检查，出门记录，出错兜底。**
