## 一句话结论

MyBatis 批量插入常用 `foreach` 拼接多行 values，也可以使用批处理执行器，重点是控制单批大小、事务和失败重试。

## 通俗解释

批量插入像一次搬一箱书。一本一本搬太慢，一次搬太多又容易摔，合理分批最稳。

## 面试回答

常见方式：

- XML 中使用 `foreach` 拼接多条 values。
- 使用 MyBatis batch executor 批量提交。
- 大数据量时分批处理，比如每 500 或 1000 条一批。

需要注意：

- 单条 SQL 不能太大，避免超过数据库限制。
- 批量操作要放在事务里。
- 失败时要明确是整批回滚还是记录失败数据后继续。
- 导入场景要做参数校验和重复数据处理。

## 常见追问

### 批量越大越好吗？

不是。太大可能导致 SQL 过长、锁时间变长、内存占用过高。

### 批量插入失败怎么处理？

看业务要求，可以整批回滚，也可以记录失败行并让用户修正后重试。

## 易错点

- 不要一次塞几十万条到一条 SQL。
- 批处理要考虑事务边界。
- 导入类业务要有失败明细和幂等处理。
