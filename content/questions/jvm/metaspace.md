## 一句话结论

元空间是 JVM 存放类元数据的区域，使用本地内存，类加载过多或类加载器无法回收可能导致 Metaspace OOM。

## 通俗解释

堆里放对象，元空间像公司档案室，放类的说明书。类越多，说明书越多，档案室也会被塞满。

## 面试回答

元空间主要保存：

- 类的结构信息。
- 方法元数据。
- 字段信息。
- 常量池相关元数据。

JDK 8 后永久代被元空间替代。元空间使用本地内存，不在 Java 堆中，但仍然可以通过参数限制大小。

如果动态生成类很多，比如频繁使用代理、反射、脚本引擎、热部署，可能导致元空间增长。

## 常见追问

### 元空间 OOM 怎么排查？

看类加载数量、类加载器数量、是否有动态代理或热部署导致类加载器无法释放。

### 元空间在堆里吗？

不在 Java 堆中，它使用本地内存。

## 易错点

- 不要把元空间和堆混为一谈。
- 元空间不是无限的，本地内存也会耗尽。
- 类加载器泄漏会导致类元数据无法释放。

## 记忆钩子

**堆放对象，元空间放类说明书。**
