## 一句话结论

逃逸分析是 JVM 判断对象是否会被当前方法或线程之外访问的优化手段，常见优化包括栈上分配、锁消除和标量替换。

## 通俗解释

逃逸分析像判断一个工具会不会被借出去。如果工具只在自己工位上用，没必要登记进公共仓库；如果会被别人拿走，就必须按更严格的方式管理。

## 面试回答

逃逸分析主要分析对象的作用范围：

- 方法逃逸：对象被返回出去，其他方法可能访问。
- 线程逃逸：对象被其他线程访问，风险更高。
- 不逃逸：对象只在当前方法内部使用。

如果 JVM 判断对象不逃逸，可能做这些优化：

1. 栈上分配：对象可能不用进入堆，方法结束就释放。
2. 锁消除：对象只被当前线程访问，同步锁可以去掉。
3. 标量替换：把对象拆成几个普通变量，减少对象创建成本。

这些优化由 JIT 在运行时根据热点代码进行，不是所有对象都会被优化。

## 常见追问

### 逃逸分析一定会让对象分配到栈上吗？

不一定。逃逸分析只是优化前提，最终是否优化取决于 JVM 实现、运行参数和代码热点情况。

### 锁消除为什么安全？

如果对象不会被其他线程访问，那么这个对象上的同步就没有竞争，JVM 可以去掉锁。

## 易错点

- 不要说所有局部对象都一定在栈上分配。
- 不要把逃逸分析当成 Java 语法特性，它是 JVM 优化。
- 面试里要能说出三类优化，不只背名词。

## 记忆钩子

**对象没逃出去，就能少管一点：少上堆、少加锁、少建对象。**
