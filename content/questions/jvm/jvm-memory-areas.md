## 一句话结论

JVM 运行时内存主要分为线程私有区和线程共享区：程序计数器、虚拟机栈、本地方法栈是线程私有；堆和方法区是线程共享。

## 通俗解释

一个 Java 程序像一家公司。每个员工都有自己的工作本和任务清单，这就是线程私有区；公司也有公共仓库和资料室，这就是线程共享区。

线程私有区跟着线程创建和销毁，线程共享区则被整个进程里的线程共同使用。

## 面试回答

JVM 运行时数据区可以这样拆：

- 程序计数器：记录当前线程执行到哪条字节码指令。
- Java 虚拟机栈：保存方法调用的栈帧，包括局部变量表、操作数栈、返回地址等。
- 本地方法栈：服务 native 方法。
- 堆：存放对象实例，是垃圾回收的主要区域。
- 方法区：存放类信息、常量、静态变量、即时编译后的代码等。HotSpot 在 JDK 8 以后用元空间实现方法区。

线程私有区域通常不会发生线程间共享问题；堆和方法区是线程共享的，所以对象分配、类加载、GC 等问题常围绕这两个区域展开。

## 常见追问

### 堆和栈有什么区别？

栈主要服务方法调用，存放局部变量、操作数等，生命周期跟线程和方法调用相关。堆主要存放对象实例，由 GC 管理。

### 方法区、永久代、元空间是什么关系？

方法区是 JVM 规范里的概念。永久代和元空间是 HotSpot 对方法区的实现方式。JDK 8 后永久代被移除，改用本地内存中的元空间。

## 易错点

- 不要把方法区和永久代完全等同，它们是规范和实现的关系。
- 不要说所有对象都一定在堆上，逃逸分析后可能有栈上分配等优化。
- 不要忘记程序计数器是线程私有且几乎不会 OOM 的区域。

## 记忆钩子

**私有三件套：计数器、虚拟机栈、本地方法栈；共享两大块：堆、方法区。**
