## 一句话结论

JVM 运行时内存主要分为线程私有区和线程共享区：程序计数器、虚拟机栈、本地方法栈是线程私有；堆和方法区是线程共享。

## 通俗解释

一个 Java 程序像一家公司。每个员工都有自己的工作本和任务清单，这就是线程私有区；公司也有公共仓库和资料室，这就是线程共享区。

线程私有区跟着线程创建和销毁，线程共享区则被整个进程里的线程共同使用。

## 面试回答

JVM 运行时数据区可以这样拆：

- 程序计数器：记录当前线程执行到哪条字节码指令。
- Java 虚拟机栈：保存方法调用的栈帧，包括局部变量表、操作数栈、返回地址等。
- 本地方法栈：服务 native 方法。
- 堆：存放对象实例，是垃圾回收的主要区域。
- 方法区：存放类信息、常量、静态变量、即时编译后的代码等。HotSpot 在 JDK 8 以后用元空间实现方法区。

线程私有区域通常不会发生线程间共享问题；堆和方法区是线程共享的，所以对象分配、类加载、GC 等问题常围绕这两个区域展开。

## 常见追问

### 这题最容易被追问的边界是什么？

重点说清堆外内存、容器限制、GC 停顿、类加载泄漏、JIT 退化和版本默认值变化。不要只给结论，要说明哪些条件下结论成立，哪些条件下会退化或需要换方案。

### 怎么证明自己不是在背模板？

用GC 日志、JFR、jcmd、jstack、jmap、NMT、CodeCache、Safepoint 和线程状态做验证。能说出具体信号、反例或排查入口，答案就从概念层进入实战层。

### 和相近方案怎么区分？

可以拿GC 参数、内存拆分、对象生命周期分析、类加载排查或运行时采样对比，从停顿时间、吞吐、内存占用、排障成本和 JDK 版本几个维度选择。

### 面试官继续深挖时怎么展开？

先围绕“程序计数器：记录当前线程执行到哪条字节码指令。”讲清主链路，再补“补 HotSpot 实现边界：方法区是规范概念，元空间主要放类元数据，JIT 编译后机器码更应落到 Code Cache”。如果能把边界条件、异常分支和验证闭环连起来，就能接住二面、三面的追问。

## 易错点

- 不要把方法区和永久代完全等同，它们是规范和实现的关系。
- 不要说所有对象都一定在堆上，逃逸分析后可能有栈上分配等优化。
- 不要忘记程序计数器是线程私有且几乎不会 OOM 的区域。

第三步是补边界。这里尤其要主动说明堆外内存、容器限制、GC 停顿、类加载泄漏、JIT 退化和版本默认值变化。不要把方法区和永久代完全等同，它们是规范和实现的关系。；不要说所有对象都一定在堆上，逃逸分析后可能有栈上分配等优化。；不要忘记程序计数器是线程私有且几乎不会 OOM 的区域。很多八股答案的问题不是方向错，而是没有说适用范围；一旦遇到数据规模变化、异常分支或版本差异，原来的结论就可能不成立。

第四步是补专项深度：补 HotSpot 实现边界：方法区是规范概念，元空间主要放类元数据，JIT 编译后机器码更应落到 Code Cache。这部分适合放在面试回答后半段，用来证明你不仅知道标准答案，也知道高频追问会往哪里挖。
